Fabric Engine 2.3.0 リリースノート¶

Fabric Engine version 2.3.0
2016年7月吉日リリース
Copyright (c) 2010-2016, Fabric Software Inc. All rights reserved.

このリリースについて¶

この Fabric Engine 2.3 のリリースでは 「Blocks」-Canvasでできることを根本から書き換えるとともに、Canvas をより簡単に扱えるようにする- を導入しました！ Blocksの詳細とその他の変更追加点については以下をご覧ください。

Blocks

これまでの Fabric についての知識のすべては Blocks により塗りつぶされます。Canvas ビジュアルプログラミングツールについて、以前は「なかなかいいんじゃないかな、でもこれでコードを書いて何かをつくるのにはちょっといかがなものかな」という印象をお持ちであったかもしれません。Blocksによってこれからは、「これはマジパネェ、全部をグラフィカルでできるよ！」という段階に引き上げます。Blocks フォルダ内に Block アーキテクチャの活用方法を示すため多数のサンプルとプリセットを同梱しています。

  • ForLoop 伝統的な Forループ

  • AccumulateWhile 配列を繰り返し処理（iterate)し、終了条件まで各処理の値を蓄積します。

  • Filter block内の条件に従い、配列をフィルタ

  • Geometry Deform ジオメトリプリミティブの頂点を繰り返し処理し、変更を加える blockを公開（expose)

  • Image Modify 画像の各ピクセルを繰り返し処理し、変更を加える blockを公開（expose)

  • AnyElementMatches 配列を繰り返し処理し、各要素について block をテスト; 要素がマッチし次第 true を戻し、なにもマッチしなければ false を戻す。

ドキュメンテーション中 Blocks について新規の章が２つあります。また What’s New の解説ビデオが概要の把握に役立つでしょう。

  • The Blocks section of the Canvas User Guide
  • The Blocks section of the Canvas Programmer Guide
  • What’s New In Fabric Engine 2.3 video

Function Node エディタ改造

Functionエディタ中、ポートの作成と変更をよりビジュアルワークフローに寄り添えるよう、メタデータの削除、ドラッグ・アンド・ドロップ、編集が可能になりました。blockを作成し、KLエディタに引きこむこともできます。

UX 改良

あたらしいFunction Nodeエディタとは別に、多くの UI/UX の改善を盛り込みました:

  • マウス中ボタンによるナビゲーションをサポート

  • ファイルメニューに最近使用したファイル履歴

  • タブによる検索ボックスウィジェットの位置を改善

  • ログウィジェットのハイライト色とフォントサイズを改善

  • ログウィジェットで自動スクロール

  • サブグラフ中からポートのデータ型を選択可能に

  • コード補完ウィンドでのコントラストを改善

  • スライダ上をクリックした際クリック位置に応じて値を設定

  • Canvas ヘッダコントロール（go back, split presets)を改善

  • グラフ外からポートを外部公開可能に、ヘッダのクリックから

Alembic 改善

Alembicエクステンションに対し多数のバグ修正をおこない、信頼性と柔軟性をより高めました。

基本的な線形代数（LA）エクステンション

ごく基礎的なライブラリで、複素数や任意の行列のコレスキー分解、二次方程式をサポートします。

Fabric for Maya の安定性向上 と Maya 2016 Extension 2 サポート

Fabric Engine 2.3.0 では数多くの Fabric for Maya の問題の修正をおこない、 Maya 2016 Extension 2 のサポートも含まれます。

サポートするOS, DCC統合, GPU¶

  • Fabric 2.3.0 は Windows, Linux をサポートします。OS X はバイナリビルドが提供されますが公式にはサポートされません。ただしこれからのリリースでサポート予定です。

  • Fabric for Maya 統合では Windows/Linux 版 Autodesk Maya 2014, 2015, 2016 （とそれぞれのサービスパック、エクステンション）をサポートします。

  • Fabric for Softimage 統合版は Autodesk Softimage 2014、2015（SP版含むWindows版）をサポートします。

  • Fabric for MODO 統合版は MODO 901 から MODO 10.0 まで Windows, Linux, OS X版を利用可能です。

  • Fabric 2 GPU supportは NVIDIAハードウェアのみ利用可能です。詳細はGPUコンピューティングガイドを御覧ください。

Fabric Engine 2.3.0 を取得するには http://dist.fabric-engine.com/FabricEngine/2.3.0/ から行えます。インストールについては Fabric Engine 入門 を参照してください。

サポート¶

Fabric 2.3.0 は評価版、スタジオ版、教育版が無償で利用可能です。また商用条件に基づく利用も可能です。商用ライセンスをご利用の顧客の皆様は、私達Fabricへ直接連絡いただけます。非商用のお客様はサポートについて以下のオプションをとることをお勧めいたします。

  • http://forums.fabricengine.com :Fabric コミュニティによる Ask & answer questions。サポートに関する問題を投稿する場です。

  • mailto://bugs@fabricengine.com : もしバグを発見したら、明快な再現手段を付しEメールをください。(使用OSとDCCアプリ、それぞれのバージョンについての詳細も)Fabricからの返答は、Fabricプラットフォームの問題の解決にあたって追加情報を必要とする場合のみですので、ご了承ください。

OS X 上での Maya 2016 Ext 2では、”OpenGL Strict” プロファイルの使用を必要とします。これは Maya 2016 からの変更であり、ビューポート上に以下エラーメッセージが表示される場合があります。”Current drawing engine does not support legacy viewport.” Fabric の Inline Drawing を正常に動作させるため、Maya の Windows Settings/Preferences ダイアログ から“OpenGL Legacy” に設定します。このオプションは Viewport 2.0 セクションの下方にあります。

変更点¶

  • canvas.bat と canvas.sh を追加、Canvasの起動が便利に

  • JSONエクステンション の API に変更がはいりました。既存のコードの挙動が変わる場合があります。

  • あたらしい配列プリセット、 Sum, Product, Average, Min, Max を追加

  • あたらしいプリセット ‘ReportInChunks’ を追加、あたらしいサンプル ‘Miscellaneous/report_in_chunks’ を追加

  • 非推奨となっていた Spliceノードのメニューを削除、Fabricメインメニューを整理。ただし Mel や Python経由で機能にアクセスすることは可能

  • Canvasスタンダロン版の既定のフレームレートを変更、 ‘max fps’ から ‘60.0 fps’ に

  • Fabric for Maya にあたらしいメニューアイテムを追加: ‘Import/Export Graph’ と ‘Select Canvas Node/Item’ Canvasに表示している Maya のCanvas ノードを選択

  • “Open Documentation” の挙動変更、まずローカルに存在するドキュメントを開こうとする。存在しない場合オンラインのドキュメントを開く。

  • GenericValue.getValueType() を変更。型をあらわす文字列ではなく、型を戻す。つまり、配列では “TypeArray” ではなく、 “Type[]” を戻す。（文字列に変換したばあい）
